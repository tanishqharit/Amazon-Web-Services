# What is Data?

## Simple Explanation

**Data** is raw facts and figures. It has no meaning by itself until we process it.

---

## Data vs Information vs Knowledge

| Concept | What it is | Example |
|---------|-----------|---------|
| **Data** | Raw facts | `25`, `"Delhi"`, `"2024-01-15"` |
| **Information** | Processed data with context | "Temperature in Delhi was 25°C on Jan 15" |
| **Knowledge** | Information + experience | "January in Delhi is usually cold, 25°C is unusually warm" |

---

## Types of Data by Structure

| Type | Description | Example |
|------|------------|---------|
| **Structured** | Organized in rows and columns | Excel spreadsheet, SQL database |
| **Semi-structured** | Has some organization | JSON, XML, CSV |
| **Unstructured** | No fixed format | Images, videos, emails, PDFs |

We'll learn each of these in detail in later notes.

---

## Types of Data by Movement

| Type | Description | Example |
|------|------------|---------|
| **Data at rest** | Stored somewhere | File in S3, row in a database |
| **Data in motion** | Being transferred | Streaming from an app to a server |
| **Data in use** | Being processed | A Glue job transforming data |

---

## Why Data Matters

- Companies use data to make **better decisions**
- Data helps find **patterns and trends**
- Data engineering is about making data **available, reliable, and fast**

---

## Key Takeaway

> Data is raw facts. Data engineers transform raw data into useful information that businesses can act on.
