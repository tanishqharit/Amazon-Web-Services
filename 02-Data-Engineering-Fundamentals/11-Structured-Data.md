# Structured Data

## What is Structured Data?

**Structured data** is data that is organized in a **fixed format** with rows and columns — like a spreadsheet or a database table.

Every piece of data has a **defined type and position**.

---

## Examples

```
| ID  | Name     | Age | City     |
|-----|----------|-----|----------|
| 1   | Tanishq  | 25  | Delhi    |
| 2   | Priya    | 30  | Mumbai   |
| 3   | Rahul    | 28  | Kolkata  |
```

Each row is a **record**. Each column has a **fixed data type** (number, text, date).

---

## Characteristics

- ✅ Easy to search and query (SQL works great)
- ✅ Well-defined schema (you know the structure in advance)
- ✅ Stored in relational databases (MySQL, PostgreSQL, Redshift)
- ❌ Not flexible for changing data structures

---

## Where You Find Structured Data

- Relational databases (RDS, Aurora)
- Spreadsheets (Excel, Google Sheets)
- Data warehouses (Redshift)
- CSV files (when they follow a fixed format)

---

## Key Takeaway

> Structured data has a fixed schema with rows and columns. It's the easiest type to query with SQL. Relational databases store structured data.
